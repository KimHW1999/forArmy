# 단계별 개발 계획

## 현재 진행 상태

- 완료: 0단계 ~ 10단계
- 현재 앱 상태: 병무청 API 연동, 특기 전체 검색, 상세 조회, 조건 입력, 추천 결과 표시, 구비서류 체크리스트, CSV 파일 캐시, 마이페이지, 기본 시연 가능
- 다음 단계: 임무 설명 데이터 보강, 추천 로직 정교화, 배포 준비
- 참고: 현재 연동한 병무청 OpenAPI에는 특기별 실제 임무 설명 본문이 없어, 공식 모집안내 자료나 별도 검수 데이터로 보강해야 한다.

## 0단계: 자료 정리

### 작업

- `API 설명.txt`와 OpenAPI 문서 기준으로 API 목록 확정
- 구비서류, 배점기준, 군사특기, 지원가능 정보, 접수현황 필드 정리
- 응답 필드 매핑표 작성
- API별 화면 사용처 정리

### 완료 기준

- 어떤 API를 어떤 기능과 화면에 사용할지 문서화되어 있다.
- 구비서류와 배점기준을 특기 상세 화면에 어떻게 묶을지 기준이 정리되어 있다.

---

## 1단계: 프로젝트 뼈대 구성

### 작업

- `apps/api`: 백엔드 서버 구조 구성
- `apps/web`: 프론트엔드 앱 구조 구성
- `packages/shared`: 프론트/백엔드 공통 타입 구성
- `prisma`: DB 스키마 위치 구성
- `.env.example` 작성

### 완료 기준

- 프론트, 백엔드, 공통 타입, DB 스키마의 책임이 분리되어 있다.
- 이후 Express, React, Prisma를 붙일 수 있는 기본 파일 구조가 준비되어 있다.

---

### 작업

## 3단계: 백엔드 조회 API 구현

### 작업

- [x] `GET /api/specialties`: 군사특기 목록
- [x] `GET /api/specialties/:code`: 군사특기 상세
- [x] `GET /api/specialties/:code/documents`: 구비서류 목록
- [x] `GET /api/specialties/:code/scores`: 선발 배점기준
- [x] `GET /api/recruitment/status`: 군지원 접수현황

### 완료 기준

- 프론트가 병무청 API를 직접 호출하지 않고 자체 백엔드 API만 호출한다.
- 특기 상세에서 구비서류와 배점기준을 함께 조회할 수 있다.

### 작업

- [x] 희망 군 일치 점수 계산
- [x] 관심 분야 일치 점수 계산

| 항목 | 점수 |
| --- | ---: |
| 희망 군 일치 | 20 |
| 관심 분야 일치 | 20 |
| 전공 조건 일치 | 20 |
| 자격/면허 조건 일치 | 20 |
| 신체 조건 충족 | 10 |
| 현재 모집 중 | 10 |

### 완료 기준

- 조건 입력 후 추천 특기 리스트가 나온다.
- 추천 사유와 부족한 조건이 함께 표시된다.

## MVP 우선순위

현실적인 1차 MVP는 다음 범위까지로 잡는다.

1. 자료 정리
2. 프로젝트 뼈대 구성
3. 병무청 API 연결
4. 백엔드 조회 API
5. 프론트 특기 목록/상세 화면
6. 조건 입력 화면
7. 추천 로직

시간이 남으면 구비서류 체크리스트까지 포함한다.
